# 测试场景

## 场景 1：新课程资料文件夹启动

用户输入：

“使用 FinaleForge Coach，基于当前文件夹资料带我交互式学习。”

Skill 应有行为：

- 扫描当前文件夹资料。
- 创建 `./交互学习输出/`。
- 先生成资料盘点、重点清单、覆盖矩阵、学习路线。
- 输出文件更新说明。
- 暂停等待用户确认。

不应该出现的错误行为：

- 直接开始讲第一个考点。
- 一次性生成完整讲义。
- 忽略老师划重点文件。

## 场景 2：索引完成后选择训练强度

用户输入：

“1”

Skill 应有行为：

- 如果当前状态是文件确认菜单，解释 1 表示继续学习。
- 进入“等待用户选择学习强度”状态。
- 输出轻量、标准、强化、自定义四种模式。

不应该出现的错误行为：

- 把 1 误解为四选项菜单中的“下一个考点”。
- 未选择强度就进入正式学习。

## 场景 3：概念型考点学习

用户输入：

“选择标准模式，开始学习。”

Skill 应有行为：

- 写入学习强度配置。
- 进入第一个未掌握考点。
- 按教学模板输出一句话核心、大白话、生活例子、学科例子、易错反例、考试版本、典型例题、随堂题。
- 出题后停止等待。

不应该出现的错误行为：

- 只给定义不举例。
- 讲多个考点。
- 出题后立刻给答案。

## 场景 4：用户答错

用户输入：

“我觉得 A 和 B 是一样的。”

Skill 应有行为：

- 批改并指出具体错因。
- 给标准答案和原因。
- 记录到错题本。
- 更新学习进度表。
- 补讲更小一步。
- 给更基础的同类题。

不应该出现的错误行为：

- 只说“错了”。
- 不记录错题。
- 直接进入下一个考点。

## 场景 5：用户中途插问

用户输入：

“这个知识点考试一般怎么给分？”

Skill 应有行为：

- 进入插问处理中。
- 简短回答。
- 如有必要记录到插问停车场。
- 明确回到当前考点、当前题目或四选项。

不应该出现的错误行为：

- 切换成普通聊天。
- 忘记当前题目。
- 自动进入新考点。

## 场景 6：考点完成

用户输入：

连续答对达到通关条件。

Skill 应有行为：

- 更新进度表和通关记录。
- 输出四选项菜单：
  1. 继续学习下一个考点
  2. 强化当前考点
  3. 复盘归纳当前考点
  4. 生成当前考点的一组测试题
- 等待用户选择。

不应该出现的错误行为：

- 自动进入下一个考点。
- 不给强化或复盘选项。
- 用户回复非数字时直接执行。

## 场景 7：错题到期复习

前置条件：

`05_错题本.md` 中存在 `下次复习日期 <= 今天` 且 `复习状态` 不是“已巩固”的错题。

Skill 应有行为：

- 在进入下一个新考点前提示有到期错题。
- 插入最多 3 道错题回顾。
- 每次仍然只出 1 道题并等待用户回答。
- 批改后更新已复习次数、下次复习日期、复习间隔和复习状态。

不应该出现的错误行为：

- 只记录错题不安排复习。
- 一次性把错题答案全部给出。
- 用错题回顾覆盖正常学习路线。

## 场景 8：用户启动错题专项

用户输入：

“复习错题本”

Skill 应有行为：

- 进入错题复习专项模式。
- 读取 `05_错题本.md`。
- 按到期优先、A 级优先、错因频率高优先排序。
- 每次只给 1 道错题或同类变式题。

不应该出现的错误行为：

- 继续推进新考点。
- 不更新错题本复习调度字段。

## 场景 9：用户要求调整优先级

用户输入：

在文件确认阶段回复“4”，并说明“把 C003 改成 A 级”。

Skill 应有行为：

- 确认当前状态是等待用户确认资料文件。
- 修改 `01_重点原始清单.md`、`02_考点覆盖矩阵.md`、`03_交互学习路线.md`。
- 在路线调整记录中写明用户指定和原因。

不应该出现的错误行为：

- 把 4 当成通关菜单的“生成小测”。
- 只口头答应但不更新文件。

## 场景 10：文件夹为空或无法读取资料

前置条件：

当前文件夹为空，或所有文件都无法读取。

Skill 应有行为：

- 生成或说明资料盘点风险。
- 明确无法抽取考点。
- 询问用户是否提供可读取资料或指定核心文件。

不应该出现的错误行为：

- 编造考点。
- 直接开始正式学习。

## 场景 11：PDF 是扫描图片

前置条件：

PDF/PPT 主要是图片扫描，无法提取文字。

Skill 应有行为：

- 在无法读取文件清单中标注“疑似扫描件/图片型文档”。
- 说明需要 OCR、截图识别或用户提供文字版。
- 不把无法读取文件当作已覆盖内容。

不应该出现的错误行为：

- 假装已经读取内容。
- 把覆盖矩阵标成已覆盖。

## 场景 12：多门课资料混在一起

前置条件：

当前文件夹同时出现多门课程资料。

Skill 应有行为：

- 先按文件名、目录、内容线索列出可能课程分组。
- 要求用户选择本次复习哪一门课。
- 用户选择前不生成混合考点路线。

不应该出现的错误行为：

- 混合不同课程重点。
- 用一套进度表追踪多门课。

## 场景 13：等待答题时用户退出

用户输入：

“退出交互学习”

Skill 应有行为：

- 将当前状态改为已结束学习。
- 更新 `06_当前学习状态.md`。
- 不继续批改、不继续出题。

不应该出现的错误行为：

- 把退出当作题目答案批改。
- 继续追问同一道题。

## 场景 14：资料全英文

前置条件：

课程资料主要是英文。

Skill 应有行为：

- 保留英文原术语。
- 用中文讲解和批改。
- 在重点清单中保留“英文关键词”和“中文提炼考点”。

不应该出现的错误行为：

- 错译核心术语。
- 删除英文原文关键词。

## 场景 15：通关后选择生成测试题

前置条件：

当前状态是“等待用户选择考点完成后的四选项”。

用户输入：

“4”

Skill 应有行为：

- 判断 4 表示“生成当前考点的一组测试题”。
- 默认生成 3 题：基础题、典型题、变式题。
- 不直接给答案。
- 一次只要求用户作答 1 题。
- 完成 3 题后回到四选项菜单。

不应该出现的错误行为：

- 把 4 当作文件确认阶段的优先级调整。
- 一次性给出题目和答案。
- 完成测试题后自动进入下一个考点。

## 场景 16：自定义错题回顾策略

用户输入：

“自定义模式：每个考点 2 题，连续答对 2 题，答错追加 1 题，错题每两个考点前最多 3 题。”

Skill 应有行为：

- 解析最少题数、连续答对数、答错追加题数和错题插入策略。
- 复述配置并要求用户确认。
- 用户确认后写入 `07_学习强度配置.md`。

不应该出现的错误行为：

- 未确认就进入正式学习。
- 要求用户输入考试日期或启用时间压力模式。

## 场景 17：插问停车场回收

前置条件：

`08_插问停车场.md` 中存在“是否需要后续展开”为“是”的记录，当前考点刚通关。

Skill 应有行为：

- 在四选项菜单前或菜单旁提示是否处理待展开插问。
- 用户选择处理时，每次只处理 1 条。
- 处理后标记为已回答，并回到原来的四选项菜单。

不应该出现的错误行为：

- 永久忽略待展开插问。
- 在用户还没答完题时主动展开插问。

## 场景 18：学科题型适配

前置条件：

当前考点是 SQL 操作题、数学计算题、英语翻译题或理论证明题之一。

Skill 应有行为：

- 根据当前题型调整讲解和批改重点。
- SQL 题关注语法和执行逻辑；计算题关注步骤和结果；语言题关注语法和表达；证明题关注推理链。
- 仍然保持一个小考点一轮，不扩展成完整讲义。

不应该出现的错误行为：

- 对所有课程使用同一种概念背诵模板。
- 为了适配题型而引入未学知识点。

## 场景 19：错题专项中途回到正常路线

前置条件：

当前处于错题专项模式。

用户输入：

“继续学新考点”

Skill 应有行为：

- 如果没有正在等待回答的错题，结束错题专项，更新 `06_当前学习状态.md`，回到正常学习路线。
- 如果已经展示错题且正在等待回答，先确认是否放弃当前错题。
- 用户确认放弃后，不把该题记为新错题，保留原错题复习状态。
- 回到正常路线前检查是否有待展开插问。

不应该出现的错误行为：

- 把“继续学新考点”当作错题答案批改。
- 强迫用户必须完成整个错题专项才能返回主线。
- 放弃当前错题时新增一条错误记录。

## 场景 20：文件确认阶段查看文件后回流

前置条件：

当前状态是“等待用户确认资料文件”。

用户输入：

“3，查看 01_重点原始清单.md”

Skill 应有行为：

- 展示或摘要指定文件。
- 状态仍保持“等待用户确认资料文件”。
- 查看完成后重新显示文件确认菜单：1 继续学习、2 修改文件、3 查看文件、4 调整优先级。
- 用户未回复 1 前，不进入学习强度选择。

不应该出现的错误行为：

- 查看文件后直接进入正式学习。
- 查看文件后直接进入训练强度选择。
- 忘记重新显示文件确认菜单。
