# 状态机与输入规则

## 交互学习状态机

Skill 在任何时候必须处于以下状态之一：

1. 资料索引中
2. 等待用户确认资料文件
3. 等待用户选择学习强度
4. 正在讲解当前考点
5. 等待用户回答题目
6. 正在批改用户答案
7. 等待用户选择考点完成后的四选项
8. 插问处理中
9. 暂停学习中
10. 已结束学习

每次回复前先判断当前状态，并同步 `./交互学习输出/06_当前学习状态.md`。

## 用户输入识别

| 用户输入 | 处理规则 |
|---|---|
| 开始 / 开始学习 | 若未选择训练强度，先要求选择；若已选择，从当前未掌握考点开始 |
| 继续 | 若等待四选项，提示回复 1/2/3/4；若无未完成题目，进入下一个应学考点 |
| 1 | 在四选项中表示继续下一个考点；在文件确认菜单中表示继续学习 |
| 2 | 在四选项中表示强化当前考点；在文件确认菜单中表示修改文件 |
| 3 | 在四选项中表示复盘当前考点；在文件确认菜单中表示查看文件 |
| 4 | 在四选项中表示生成小测；在文件确认菜单中表示调整考点优先级 |
| 我没懂 / 换个说法 / 再讲一遍 | 不进入新考点，换更简单例子重讲当前考点 |
| 这题不会 / 给点提示 | 不给完整答案，只给一步提示，继续等待回答 |
| 暂停 / 先不学了 | 更新当前学习状态，进入暂停学习中 |
| 退出交互学习 | 进入已结束学习 |
| 复习错题本 | 进入错题复习专项，按错题本调度出题 |
| 继续学新考点 / 退出错题专项 | 若正在错题专项，结束专项并回到正常学习路线；若正在等待题目答案，先确认放弃当前错题 |

必须根据当前状态解释数字含义，不能把文件确认菜单的 `1` 和通关四选项的 `1` 混用。

## 等待状态约束

- 当前状态是“等待用户回答题目”时，除非用户明确暂停或退出，不得进入新考点。
- 当前状态是“等待用户选择考点完成后的四选项”时，用户必须回复 1、2、3、4。若用户插问，先简短回答，再回到四选项。
- 当前状态是“等待用户确认资料文件”时，必须允许用户回复 4 调整某个考点的优先级。

## 错题专项退出

用户在错题专项中可以主动说“继续学新考点”或“退出错题专项”回到正常路线。

处理规则：

1. 如果当前没有等待用户回答的错题，立即结束错题专项，更新 `06_当前学习状态.md` 中“当前是否处于错题专项”为“否”，再按 `03_交互学习路线.md` 找下一个应学考点。
2. 如果当前已经展示错题并正在等待回答，先确认：“当前错题还没回答，要放弃这题并继续新考点吗？回复 1 放弃并继续，回复 2 先完成这题。”
3. 用户回复 1 后，不把该题记为新错题；保留原错题复习状态，回到正常学习路线。
4. 用户回复 2 后，继续等待当前错题答案。
5. 退出错题专项后，按插问停车场回收规则检查是否有待展开插问。

## 插问回归和防漂移

一旦进入交互式学习模式，用户中途提问视为插问，不是退出学习。

处理步骤：

1. 简短回答问题。
2. 如果和当前考点无关，记录到 `08_插问停车场.md`。
3. 明确回到当前考点、当前题目或当前四选项菜单。
4. 继续等待用户完成当前学习动作。

固定格式：

【插问解答】

【回到当前学习】

我们刚才还在学习：xxx  
当前进度是：xxx  
现在继续：xxx

## 插问停车场回收

`08_插问停车场.md` 中“是否需要后续展开”为“是”的条目不能永久搁置。

检查时机：

1. 当前考点通关并进入四选项菜单时。
2. 错题专项模式结束时。
3. 用户主动说“处理插问停车场”时。

处理规则：

1. 若存在待展开插问，先提示：“插问停车场里还有待展开问题，要现在处理吗？回复 1 处理，回复 2 稍后。”
2. 用户回复 1 时，每次只处理 1 个插问；处理完标记为已回答，并回到原来的四选项或学习状态。
3. 用户回复 2 时，保留停车场记录，不影响继续学习。
4. 如果当前仍在等待用户回答题目，不能主动展开插问，只能在题目完成后检查。

## 回复前自检清单

1. 我现在是否还在交互式学习模式？
2. 当前状态是什么？
3. 当前考点是什么？
4. 是否有题目正在等待用户回答？
5. 是否不小心提前给出了答案？
6. 是否使用了未学习知识点作为关键依据？
7. 是否需要更新学习进度表、错题本或当前学习状态？
8. 如果刚完成考点，是否给出了 1、2、3、4 四选项？
